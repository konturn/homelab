## 12:05-13:40 — JIT Approval Service Deep Work Session

Major session building out the JIT approval service with Noah. Key developments:

### Architecture Walkthrough
- Walked Noah through the full Radarr approval flow (Tier 1, auto-approve, 15-min Vault token)
- Explained the gap: Vault tokens gate *access to secrets*, not the secrets themselves. Most service credentials (Radarr API key, etc.) are long-lived and don't expire with the Vault token.

### Dynamic Credential Backends (Major Feature)
Noah asked if we could make Vault generate real temp tokens for services that support it. Research found 4 services support dynamic credentials:
- **Home Assistant** — OAuth2 refresh token → 30-min access tokens
- **Grafana** — Service account API → tokens with expiration dates  
- **Plex** — Transient token API → 48h tokens
- **InfluxDB** — Create/delete authorization tokens

Built and pushed full dynamic backend system:
- `internal/backend/` package with Backend interface + registry
- 4 dynamic implementations + static fallback
- Graceful fallback: if dynamic fails, falls back to static Vault token
- Removed mqtt/cameras from resource map per Noah

### Vault Secret Population
Populated all missing Vault secrets for dynamic backends:
- **InfluxDB**: org_id = `545869d8522fa417` (org name: "homelab")
- **Grafana**: Created service account ID 3 ("jit-credential-broker", Admin role), generated `jit_admin_token` (glsa_aCg...)
- **Plex**: Already had `token` field, just needed code alignment
- **Home Assistant**: Did full OAuth2 flow with Noah — he authorized via browser, I exchanged the code for refresh_token. Stored in Vault.
